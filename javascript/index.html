JavaScript

前言

现在的前端简直是发展的太快了，从最开始的jQuery作为后端附属的开发模式，到前后端分离，再到前端工程化，直到现在前端完全独立。其发展是日新月异。新的设计模式和框架举不胜举，每一种框架都解决了特定范围的问题，而且感觉前端很爱重新造轮子，这使得前端得以不断进化和升级，特别是NodeJS的出现，使得JavaScript不再只是前端，它也可以用来编写服务端逻辑。

正因为上述背景，作为一个新手，当想要开始学习前端技术的时候，面对浩瀚的代码库，其实是很难下手的，我也是这样，所以我尝试使用最原始和轻松的方式来学习这些前端技术。

在我写作的时候，jQuery已经是昨日黄花，前端已经进入了React、Vue和AngularJS三分天下的时代，本文将以React为主线来学习JavaScript，本文仅是个人学习的记录，错误是难免的，欢迎大家随时在评论区给我留言。

引入ES6

目前我们是可以直接在HTML上写JS脚本，这些脚本是遵循ES5规范的，我们都知道JS脚本是运行在用户安装的浏览器的内核上面的，各个浏览器厂家在实现浏览器内核时就被要求必须要能够理解和解释ES5的JS语法，那有了ES5为啥还要搞个ES6呢？原因就在于ES5规范搞得不好呗，所以需要更新换代。那一批来自浏览器厂家的牛人和规范制订者就说，那我们重新搞个JS的规范吧，以后大家做浏览器都要遵循这个规范，这一帮人就重新做好了规范，浏览器厂家也想遵循这个规范来实现浏览器软件，但是必须面对的一个问题就是，现在的互联网已经存在海量的基于ES5老版本的Web应用，要让所有这些应用全部改造升级到ES6那是万万不可能的，所以这个事情就有点麻烦了。那新的规范又还是挺好的，语法也更简洁明快，有不少的开发者都想用，特别是NodeJS出现以后，NodeJS直接拿ES6作为运行环境的编程语言，这大大推进了ES6的普及，而且大家想出了一个办法，拿ES6的语法写代码，然后再找工具把ES6的代码编译成ES5格式的JS脚本，这样不就能充分利用ES6的优点又能向下兼容了莫。

现在更是出现了TypeScript（TS），TS比ES6更进一步，目的在于从语法层面强制让开发者遵循约定的规范，避免出现歧义性代码，但是不管TS还是ES6，在浏览器端最终都会被编译成ES5的脚本运行。
